# Pipeline: Machine Learning (PyTorch Surrogate Model)

This directory contains the object-oriented Machine Learning pipeline designed to bridge the quantum microscopic scale (TB-DFT) with the macroscopic fluid dynamics scale (Navier-Stokes). By employing a Multi-Layer Perceptron (MLP) within the PyTorch framework, the system learns the non-linear mapping $f_{\text{ML}}: \mathcal{X} \to \mathcal{Y}$ from local lattice deformations to the kinematic viscosity $\nu$ of the electron fluid.

The architecture is strictly compliant with ISO/IEC software engineering standards, enforcing the Single Responsibility Principle (SRP), modularity, and strict separation of configuration from execution logic.

---

## 1. Module Architecture

### `config_parser.py`
**Responsibility:** Configuration Management.
Safely loads and parses the YAML configuration file (`../configs/ml_model/mlp_config.yaml`). It ensures that hyperparameters, paths, and hardware settings are fully decoupled from the core execution logic, allowing for reproducible and version-controlled experiments.

### `data_loader.py`
**Responsibility:** Data Ingestion and Preprocessing.
Acts as the physical bridge between the Fireball output and the ML model. 
* Parses `summary.csv` to extract quantum parameters: lattice constant $a$, Fermi velocity $v_F$, and Density of States $D(E_F)$.
* Mathematically evaluates the theoretical kinematic viscosity $\nu$ using the Chapman-Enskog expansion.
* Performs rigorous **Z-score Normalization** on both the feature space $\mathcal{X}$ and target space $\mathcal{Y}$ to stabilize the Empirical Risk Minimization (ERM) and prevent exploding gradients.
* Packages the data into a PyTorch `Dataset`.

### `model.py`
**Responsibility:** Hypothesis Class Definition.
Defines the `ViscosityMLP` architecture inheriting from `torch.nn.Module`. Constructs a dynamic feedforward neural network based on the hidden layer dimensions specified in the configuration. Uses Rectified Linear Unit (ReLU) activations to capture complex, non-linear physical phenomena.

### `trainer.py`
**Responsibility:** Empirical Risk Minimization (Training Loop).
Encapsulates the entire optimization procedure. 
* Manages forward and backward propagation steps.
* Applies $L_2$ Tikhonov regularization (weight decay) via the optimizer.
* Evaluates the model continuously on a validation split to track Mean Squared Error (MSE) and the Coefficient of Determination ($R^2$).
* Safely exports model weights (`.pth`) and training histories (`.csv`).

### `main.py`
**Responsibility:** Orchestration.
The primary entry point of the pipeline. It initializes the logging framework, allocates hardware resources (automatically routing to CUDA GPUs if available), instantiates the dataset, model, and optimizer, and triggers the `MLPTrainer`.

### `plot_learning_curves.py`
**Responsibility:** Statistical Validation and Visualization.
Reads the `training_metrics.csv` generated during the training phase. Produces academic-grade, publication-ready dual-axis plots demonstrating the exponential decay of the MSE and the asymptotic convergence of the $R^2$ score towards unity.

### `generate_heatmaps.py`
**Responsibility:** Physics-Informed Inference.
Validates the surrogate model in a physical context. It defines the macroscopic geometries (e.g., straight channel, small pocket, large pocket), computes local strain concentrations at the boundaries, translates this strain into local lattice constants, interpolates the corresponding quantum features, and queries the trained ML model. The output is a high-resolution 2D spatial heatmap of the kinematic viscosity $\nu(x,y)$.

## 2. Execution Workflow

To ensure physical consistency and mathematical stability, the scripts must be executed in the following chronological order.

### Step 1: Configuration Verification
Ensure that your network hyperparameters and data paths are correctly specified in `../configs/ml_model/mlp_config.yaml`. Verify that `summary.csv` has been fully generated by the `pipeline_fireball` module.

### Step 2: Model Training
Execute the main orchestrator to train the surrogate model. This step requires the physical data to be present.
```bash
python main.py
```
* **Output:** Generates `pipeline_execution.log`, `training_metrics.csv`, and the optimized network weights `viscosity_mlp.pth` in the `../data/data_learning/` directory.

### Step 3: Diagnostic Validation
Once training is complete, verify the learning stability and absence of generalization gaps (overfitting).
```bash
python plot_learning_curves.py
```
* **Output:** Generates `mlp_learning_curves.png` in `../data/data_learning/visualizations/`. An $R^2$ score $> 0.90$ is required to proceed safely to macroscopic simulations.

### Step 4: Spatial Heatmap Generation
Utilize the trained model to project the kinematic viscosity onto the spatial domain of the mesoscopic devices.
```bash
python generate_heatmaps.py
```
* **Output:** Generates three spatially resolved viscosity fields (`heatmap_no_pocket.png`, `heatmap_small_pocket.png`, `heatmap_large_pocket.png`) in `../data/data_learning/heatmaps/`. These fields act as the fundamental proof-of-concept linking strain-induced lattice deformation to hydrodynamic boundary constraints.
